# План: Фаза B — семантика стрелок в диаграммах

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. Спека: `docs/superpowers/specs/2026-06-12-diagram-palette-unification-design.md` (раздел «Словарь категорных стрелок» + кит маркеров). Инвентаризация проведена 3 агентами — результаты ниже встроены.

**Goal:** Пометить стрелки, которые семантически являются mono/iso/nat, правильными формами (`↣`/`≅`/`⇒`) в тех диаграммах, где это математически верно и нарисовано стрелкой.

**Architecture:** Точечная ручная правка SVG. Mono — хвостик `marker-start` (orient=auto сам ориентирует). Nat — двойная линия + существующий наконечник. Iso — стрелка + символ `≅`. Цвета — из канона Фазы A; ASCII и палитра не меняются. `normalize_palette.js` должен остаться идемпотентным.

**Tech Stack:** ручное редактирование SVG; визуальная проверка — в Jupyter (контейнер поднят).

**Важно:** ни один файл не использует кит `#ah*` — у каждого свои локальные маркеры. Поэтому mono-хвостик добавляем как НОВЫЙ маркер в `<defs>` файла (по образцу кита `ah-tail`), перекрашенный под цвет конкретной стрелки.

**Установлено инвентаризацией:** эпиморфизмов нет нигде; почти все `≅` — текстовые формулы (не трогаем); реальных nat-стрелок — только η/ε в adj_adjunction.

### Task 1: Мономорфизмы в топосах (5 файлов)

Метод для каждого файла: в `<defs>` добавить маркер-хвостик (по образцу), затем на нужные `<line>` добавить `marker-start="url(#motail-XXX)"`, где XXX — суффикс под цвет стрелки. Образец маркера (цвет = stroke конкретной стрелки):

```xml
<marker id="motail-ind" markerWidth="10" markerHeight="8" refX="1" refY="3.5" orient="auto" markerUnits="userSpaceOnUse"><path d="M8,0 L1,3.5 L8,7" fill="none" stroke="#3a4a96" stroke-width="1.6"/></marker>
```

(Суффиксы: `-ind` #3a4a96 индиго-узел, либо под фактический stroke стрелки — посмотреть в файле и подобрать; если стрелка нейтральная, `#8a93a3`.)
