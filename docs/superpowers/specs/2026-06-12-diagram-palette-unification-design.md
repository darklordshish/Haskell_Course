# Дизайн: унификация палитры диаграмм (светлая тема)

**Дата:** 2026-06-12. **Статус:** утверждено (палитра и решения согласованы в ревью).

## Контекст и цель

Аудит 96 SVG показал: набор **уже светлый** (тёмного фона — 0; `#0f172a` используется как
цвет текста). Документация (`DesignShowcase.ipynb`, палитра в `ROADMAP.md`) ошибочно
описывает «тёмную тему» — устарела. Реальная проблема — **фрагментация светлой палитры**:
230 различных hex, синий в 7 вариантах, зелёный расщеплён `#059669`↔`#34d399`, отдельный
навстиль `kan/` (`#334466`/`#2255cc`), выпадающие тёмно-коллажные `haskell/*_landscape`
и `topos_libraries`.

Цель: привести все диаграммы к **единой светлой палитре**, обновить доки, смягчить
правило про numeric-entities. `course_map.svg` не трогаем (у карт своя «метро»-палитра).

## Решения (ревью)

- Зелёный акцент — `#059669` (лучше контраст на белом).
- Навстиль `kan/` и оптических постеров (`#334466`) — свести к канону.
- Правило ROADMAP `&#NNNN;` — **смягчить**: numeric-entity для матсимволов это
  ASCII-безопасно и работает (ноутбуки исполняются); запрет снять, оставить запрет
  только на именованные/кириллические entity.
- Паттерн «сплошной цветной узел + белый текст» на белом фоне — легитимен, сохраняем;
  нормализуем только конкретные оттенки.

## Канон палитры

Направление — **«в тон карте курса»** (выбор ревью): приглушённая палитра, общая с
`course_map.svg`, диаграммы и карта становятся одной визуальной семьёй. Уходим с
saturated-tailwind на muted-семейство.

**Нейтрали:**

| Роль | HEX |
|------|-----|
| фон | `#ffffff` |
| карточка/светлая заливка | `#f7f8fa` |
| граница | `#e2e8f0` |
| стрелки / вторичные линии | `#8a93a3` |
| текст вторичный | `#475569` |
| текст основной | `#0f172a` |

**Акценты — приглушённые, по 3 тона на семейство** (узел = тёмный под белый текст;
база = обводка/линия/акцентный текст; тинт = светлая заливка под тёмный текст). Базовые
тона совпадают с линиями `course_map`:

| Семейство | узел | база | тинт |
|-----------|------|------|------|
| индиго (синий) | `#3a4a96` | `#5872c9` | `#eef1fb` |
| шалфей (зелёный) | `#2f6f54` | `#3f9d77` | `#e9f4ee` |
| лаванда (фиолет.) | `#5f4a86` | `#8b6db5` | `#efeaf6` |
| охра (жёлт./золото/выдел.) | `#8a6418` | `#d9a441` | `#f4e7cf` |
| терракота (красный/«нет») | `#8f3f3c` | `#c4625f` | `#f5e3e2` |

Пять семантических семейств (вместо 6 saturated): охра покрывает прежние «оранжевый +
амбер/золото», терракота — «красный/отрицание». Это ровно палитра карты, расширенная
до тёмного/тинтового тонов.

**Стиль узла (приём C, по содержимому):**
- Разрежённые объект-стрелочные диаграммы с короткими подписями — **узлы-кольца**
  как станции `course_map` (белая заливка + цветная обводка base, подпись рядом/под).
- Плотные узлы с текстом внутри (таблицы, решётки, многословные блоки) — **скруглённые
  плашки** в тон: заливка-тинт + обводка-база + тёмный цветной текст (узел-тон — только
  где нужен белый текст). Кольца туда не форсируем.

**Словарь категорных стрелок** (наконечники — полигоны в линии, не `<marker>`-зависимы):

| Морфизм | Форма | Обозначение |
|---------|-------|-------------|
| общий морфизм | одинарный наконечник | `→` |
| мономорфизм | хвостик-шеврон в начале + наконечник | `↣` (инъекция) |
| эпиморфизм | двойной наконечник | `↠` (сюръекция) |
| изоморфизм | наконечник + `≅` над линией | `A ≅ B` |
| ест. преобразование | двойная линия + наконечник | `⇒` |

Цвет стрелки — нейтраль `#8a93a3` по умолчанию либо база семейства по смыслу.
Применяется только там, где морфизм реально mono/epi/iso/nat — не на каждой стрелке.

**Кит маркеров (переиспользуемый, в `<defs>` каждого нового SVG).** Метки-`<marker>`
с фиксированными id — родное средство SVG, рисуются в Jupyter (в чат-превью `<marker>`
вырезается — это ограничение превью, не курса):

```xml
<defs>
  <marker id="ah" markerWidth="10" markerHeight="8" refX="8" refY="3.5" orient="auto" markerUnits="userSpaceOnUse"><path d="M0,0 L9,3.5 L0,7 z" fill="#8a93a3"/></marker>
  <marker id="ah-epi" markerWidth="15" markerHeight="8" refX="13" refY="3.5" orient="auto" markerUnits="userSpaceOnUse"><path d="M0,0 L9,3.5 L0,7 z M5,0 L14,3.5 L5,7 z" fill="#8a93a3"/></marker>
  <marker id="ah-tail" markerWidth="10" markerHeight="8" refX="1" refY="3.5" orient="auto" markerUnits="userSpaceOnUse"><path d="M8,0 L1,3.5 L8,7" fill="none" stroke="#8a93a3" stroke-width="1.6"/></marker>
</defs>
```

| Морфизм | Как рисовать |
|---------|--------------|
| `→` общий | `marker-end="url(#ah)"` |
| `↣` mono | `marker-start="url(#ah-tail)" marker-end="url(#ah)"` |
| `↠` epi | `marker-end="url(#ah-epi)"` |
| `≅` iso | `marker-end="url(#ah)"` + текст `≅` над линией |
| `⇒` nat | две параллельные линии + `marker-end="url(#ah)"` |

Наконечники по умолчанию нейтральные; где нужен цветной — перекрашенная копия маркера.
Кит хранится готовым блоком в шапке-правилах (`DesignShowcase.ipynb`) и упомянут в ROADMAP;
в новый SVG просто копируется в `<defs>`. Спот-чек стрелок — в Jupyter или временной
инлайн-копией наконечника (полигоном).

## Подход к нормализации

`src/scripts/coursemap/normalize_palette.js` — детерминированный реколор всех
`src/diagrams/**/*.svg` (кроме `course_map.svg`):

1. Для каждого hex: перевод в HSL.
2. **Нейтраль** (saturation < ~15%): снап к slate-шкале по светлоте
   (bg/card/border/arrow/text-2/text-1).
3. **Цветной**: бакет по hue → {индиго, шалфей, лаванда, охра, терракота};
   внутри семейства снап к {узел, база, тинт} по светлоте. Оранжевый и амбер из
   старых диаграмм сходятся в охру; алый/красный — в терракоту.
4. **Таблица override** для спорных hex, где hue/яркость дают неверный тон
   (например `#334466` — навстиль `kan`: как заливка-узел → индиго-узел `#3a4a96`;
   как стрелка/текст → `#8a93a3`; разрешается пофайловым контекстом или явным правилом).
5. Numeric-entity (`&#NNNN;`) не трогаются. ASCII-инвариант сохраняется (guard).
6. Идемпотентность: повторный прогон не меняет файлы (канонические hex — неподвижные точки).

**Файлы-исключения (ручная переделка, не скриптом)** — тёмные коллажи с инверсной
семантикой, где автоснап сломает читаемость:
`haskell/gpu_landscape.svg`, `haskell/conc_landscape.svg`, `haskell/dist_landscape.svg`,
`topos/topos_libraries.svg`. Их перерисовать под канон индивидуально.

## Фазирование (две независимые задачи)

**Фаза A — палитра (массовая, скриптуемая).** Нормализация цвета всех SVG
скриптом + 4 ручных файла + доки. Высокое покрытие за проход, проверяется
объективно (hex сошлись, фон светлый, ASCII). Это основной объём, делается первой.

**Фаза B — семантика стрелок (ручная, курируемая).** Применение словаря
`↣`/`↠`/`≅`/`⇒` там, где морфизм действительно mono/epi/iso/nat. Скриптом
*нельзя* — нужна математика каждой диаграммы. Делается отдельным проходом по
короткому списку диаграмм-кандидатов, где такие морфизмы есть:
- топосы: подобъект `S ↣ Ω` (`topos/omega_pullback`, `topos_axioms`, `three_faces`),
  факторы/сюръекции;
- Йонеда: изоморфизмы `≅` (`yoneda/yoneda_iso`, `yoneda_hom`, `yo_yoneda`);
- сопряжения/функторы: `⇒` для ест. преобразований (`adjunctions/*`, `functors/*`,
  `kan/*` единицы/коединицы);
- алгебры: уникальный морфизм катаморфизма (часто пунктир, не mono/epi).
Точный список кандидатов уточняется при планировании Фазы B; на каждую — ручная
правка с пониманием смысла. Прочие стрелки остаются обычными `→`.

## Верификация

1. `normalize_palette.js` идемпотентен (повторный прогон — `git diff` пуст).
2. ASCII-инвариант: во всех SVG только ASCII (скрипт-проверка как в build_map).
3. Тёмного фона нет (assert: ни одного крупного `<rect>` с тёмной заливкой).
4. Палитра сошлась: число различных hex резко упало (цель — 6 нейтралей + 15 акцентных
   тонов (5 семейств × 3) + белый/чёрный), остаток вне канона распечатать для ручной проверки.
5. **Визуальный спот-чек**: отрендерить по 1–2 представителя из каждой папки
   (до/после), глазами подтвердить читаемость; отдельно — 4 ручных файла и `kan/`.
6. Ноутбуки не перепрогоняются (SVG — внешние файлы, код не меняется).
