# KanExtensions.ipynb — переработка в развёрнутый учебный формат

> Дата: 2026-06-10 · Статус: дизайн утверждён, ожидает реализации
> Базируется на шаблоне `2026-06-10-uncertainty-rework-design.md` (Фаза 9).

## Проблема

`src/notebooks/KanExtensions.ipynb` (24 ячейки, 9 секций) написан телеграфно:
определение + код, без мотивации, интуиции, иллюстративных примеров и границ
применимости. Плюс конкретные дефекты:

1. **Битая диаграмма:** в секции 3 ссылка `![...](kan_examples.svg)` дана без
   префикса `../diagrams/kan/` → картинка не грузится.
2. **`kan_examples.svg` нарушает ASCII-правило** (9 не-ASCII символов).
3. **Ни у одной из 7 kan-SVG нет `viewBox`** — отсюда «кривое» масштабирование/обрезка.
4. Местами код во фенсах без `haskell`, формулы в backtick'ах вместо `$...$`.

## Решение

### Единый скелет каждой секции (как в Uncertainty)

Мотивация → Идея/категорный смысл → Формализм (`$...$`) → Код+развёрнутая демо →
Пример из жизни → Границы применимости → Мостик к следующей секции.

Особый акцент для Kan: **больше содержательных иллюстративных примеров** —
конкретные функторы (`Maybe`, `[]`, `Either`), round-trip изоморфизмы, прикладной
смысл Codensity (ускорение `>>=`, DList), почему «все концепции — расширения Кана».

### Сводка (новая финальная секция, 3 канала)

- **SVG-постер** «карта расширений Кана»: Ran/Lan, их частные случаи (Yoneda = Ran Id,
  сопряжения = Ran/Lan вдоль f, Codensity/Density = Ran/Lan f f), связь с монадами.
- **markdown-таблица**: строки = конструкции (Ran, Lan, Codensity, Density, Yoneda,
  Adjunction-as-Kan), столбцы = определение / Haskell-тип / частный случай / польза.
- **текстовый абзац** единой картины.

### Диаграммы

- **Починить все 7 существующих**: добавить `viewBox` (совпадающий с width/height),
  гарантировать белый фон `#ffffff`, выровнять элементы, исправить `kan_examples.svg`
  (убрать не-ASCII → ASCII/транслит), привести к каноничному стилю.
- **Исправить битую ссылку** на `kan_examples.svg` (полный путь `../diagrams/kan/`).
- **Добавить новые**: диаграмма к секции 8 «монада из сопряжения» (`T = g∘f`,
  `f ⊣ g ⇒ монада`), к секции 9 «Codensity-оптимизация» (право-ассоциативный `>>=`,
  `O(n²)→O(n)`), и постер для сводки.

## Инварианты

- Код исполним: **0 ошибок** (правим только прозу/комментарии/демо и при нужде —
  добавляем небольшие иллюстративные примеры, проверяя исполнение).
- Делимитеры — только `$...$`.
- SVG — ASCII-текст, белый фон, `viewBox`, defs наверху, валидный XML.
- README/ROADMAP синхронны (счётчики SVG, отметка фазы).
