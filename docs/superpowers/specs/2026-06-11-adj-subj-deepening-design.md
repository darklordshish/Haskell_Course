# Фаза 14 — углубление Adjunctions + SubjectiveModeling, связи между пятью ноутбуками

> Дата: 2026-06-11 · Статус: дизайн утверждён.
> Объём (решение пользователя): глубоко Adjunctions и SubjectiveModeling; точечно
> (аудит + связи) Kan/Toposes/Uncertainty — без повторной переработки сделанного в Фазах 9–13.

## Диагноз

**Adjunctions.ipynb (32 ячейки):** интуиция/формализм развиты (таблицы, η/ε, биекция),
но по скелету не хватает: примеров из жизни, границ применимости, мостиков между
секциями; нет финальной 3-канальной сводки; не отражена связь с Kan Фазы 10
(монада из сопряжения, `g ≅ Ran f Id`).

**SubjectiveModeling.ipynb (38 ячеек):** справочно-академический стиль (определения
Пытьева с формулами) — формализм отличный, но без мотивационных подводок, интерпретаций
«зачем» и границ. Структурные баги: «Итоги» (ячейка 23) стоят ДО разделов 15–16;
главный титул (ячейка 4) — после введения (ячейки 1–3).

## Решение

### Adjunctions — добивка до полного скелета (без разрушения готового)
К каждой содержательной секции **добавить** недостающие биты: *Пример из жизни*,
*Границы применимости*, *Мостик* (и мотивацию, где её нет). Существующий текст
сохраняется — биты дописываются. Добавить финальную секцию **Сводка** (3 канала:
постер + таблица «сопряжение → что порождает» + текст) и кросс-линк на
KanExtensions §7–8 (сопряжения через Ran, монада из сопряжения).

### SubjectiveModeling — мотивационная обвязка + структура
1. Переставить главный титул в начало (титул → введение → раздел 1...).
2. Переместить «Итоги» в конец (после раздела 16) и обновить их (упомянуть 15–16).
3. К разделам 1–14 добавить короткие подводки «Мотивация/Зачем» и «Границы/Замечания»
   там, где их нет (лёгкая рука: формальный текст Пытьева не трогаем, дописываем вокруг).
4. Кросс-линки: §3 ↔ Uncertainty §9 (один аппарат, разные интерпретации);
   §13–14 ↔ Toposes T8 (строка Sub.Mod. в таблице трёх топосов).

### Точечно (Kan / Toposes / Uncertainty)
Только взаимные ссылки-мостики, чтобы пять ноутбуков читались как одна линия:
Kan §8 → Adjunctions; Toposes T6 → Kan §7; Uncertainty §9-мост → SubjMod (уже есть, проверить).
