# Toposes.ipynb T4 — внутренняя логика в стрелочной нотации

> Дата: 2026-06-10 · Статус: дизайн утверждён · точечное расширение раздела T4.

## Проблема

Раздел **T4** даёт гейтингову алгебру на Ω **поэлементно** (открытые множества: ∩, ∪,
Int(...)), но **не в стрелочной/категорной нотации** — связки не определены как морфизмы
на Ω через классификатор подобъектов. Для главы об элементарных топосах это пробел.

## Решение — расширить T4 (без нового раздела)

Добавить в T4 подраздел «Связки как классифицирующие стрелки» + исполнимый код в Set + 2
pullback-диаграммы. Открытые формулы из существующей таблицы T4 трактуются как реализация
этих стрелок в Sh(X).

### Стрелочные определения (элементарный топос)

- `true, false : 1 → Ω`, где `false = χ(0 ↪ 1)`.
- **∧** `: Ω×Ω → Ω` `= χ⟨true,true⟩` — классификатор `⟨true,true⟩ : 1 ↪ Ω×Ω` (pullback).
- **≤** `↪ Ω×Ω` — эквалайзер `(∧, π₁)`, т.е. `{(x,y) | x∧y = x}` (внутренний порядок).
- **⇒** `: Ω×Ω → Ω` `= χ(≤)`.
- **∨** `: Ω×Ω → Ω` `= χ(im[⟨true∘!,id⟩, ⟨id,true∘!⟩])` — классификатор объединения прямых истинности.
- **¬** `: Ω → Ω` `= χ(false) = (⇒) ∘ ⟨id, false∘!⟩`.
- Гейтингова сопряжённость `(−∧a) ⊣ (a⇒−)`: `x ≤ (a⇒y) ⟺ (x∧a) ≤ y`.

### Код (Set, Ω = Bool) — исполнимый

Каждая связка строится как **характеристическая стрелка** своего определяющего подобъекта
(заданного перечислением элементов), затем проверяется совпадение с классическими таблицами
истинности и выполнение гейтинговой сопряжённости. Самодостаточная ячейка (только `Bool`/списки).

### Связь с Sh(X)

Явная фраза: открытые формулы в таблице T4 (`∩`, `∪`, `Int((X\U)∪V)`, `Int(X\U)`) — это те же
стрелки `∧,∨,⇒,¬`, реализованные в топосе пучков `Sh(X)`; в `Set` (Ω=2) они вырождаются в
булевы таблицы.

### Диаграммы (`diagrams/topos/`)

- `topos_conj_neg.svg` — pullback-квадраты для ∧ (через `⟨true,true⟩`) и ¬ (через `false`).
- `topos_impl_disj.svg` — ⇒ (через эквалайзер `≤`) и ∨ (через образ объединения прямых истинности).
