# Дизайн: ноутбук-словарь двойственности (Duality.ipynb)

**Дата:** 2026-06-16. **Статус:** согласовано в брейншторме.

## Цель

Отдельный ноутбук-справочник «парад двойственных категорных конструкций». Пользователь
дополняет его со временем. Организующая идея — **одна двойственность**: обращение стрелок
`C ⇄ C^op`. Подача: разминка на **примитивах** (обращённые стрелки) → рано вводится
**движок — расширения Кана** → и дальше всё (общие пределы, пределы над формой `J`,
сопряжения, монады/комонады) выводится **через Кан**. Слоган «всё — расширение Кана» —
рабочий метод по ходу, а не лозунг.

## Ключевые решения (брейншторм)

- **Размещение:** отдельный `src/notebooks/Duality.ipynb`, вне линейного ствола 0–VI
  (приложение/справочник). В NAV-цепочку не вставляется; ссылки — из релевантных тем
  и из README (как «приложение»).
- **Единая ось — `C^op`.** Co-приставка = «то же в `C^op`». Порядок: примитивы → Кан-движок →
  всё через Кан. Lan/Ran — зеркальная пара под этой двойственностью
  (`(Lan_K F)^op ≅ Ran_{K^op}(F^op)`) и универсальный движок: пределы = Кан вдоль `J→1`,
  сопряжения = `Ran_F Id`/`Lan_G Id`, монада коплотности = `Ran f f`, комонада плотности =
  `Lan f f`.
- **Что НЕ через Кан (честная граница):** моно/эпи, ко/контравариантный функтор,
  алгебры/коалгебры — чистые op-дуалы (обращённые стрелки / неподвижные точки эндофунктора),
  Каном не порождаются; стоят до Кан-движка как примитивы.
- **Глубина раздела:** полный — мат-определение + универсальное свойство + парная диаграмма
  (обращение стрелок/конуса) + Haskell обоих членов + строка «в чём двойственность» +
  ссылки «где в курсе».
- **Оглавление:** сводная таблица **вручную** в v1 (генератор — позже, если пар станет много).
- **v1:** каркас (мотивация + таблица + setup/NAV) + несколько наполненных разделов-образцов;
  при этом в плане прописан **скелет каждого раздела**, разделы независимы и возобновляемы.

## Структура ноутбука (примитивы → Кан-движок → всё через Кан)

0. **Мотивация** (лёгкая): двойственность = обращение стрелок (`C^op`); приставка «co-».
1. **Примитивы и структурные op-дуалы (вне Кана):** моно/эпи, ковариантный/контравариантный
   функтор, алгебры/коалгебры — чистая `C^op`-двойственность; Каном не порождаются.
2. **Движок — расширения Кана:** `Lan` (левое) ↔ `Ran` (правое), зеркало `C^op`
   (`(Lan_K F)^op ≅ Ran_{K^op}(F^op)`). Универсальное свойство, Haskell обоих.
3. **Общие пределы/копределы через Кан:** `lim = Ran` вдоль `J→1`, `colim = Lan` вдоль `J→1`.
4. **Пределы над конкретной формой `J`** (частные случаи §3) — **центральный приём**:
   форма-диаграмма `J` в центре, копредел слева (коконус), предел справа (конус). По нарастанию
   формы: терминальный/начальный (`J=∅`) → произв./копроизв. (`J`=дискретная пара) →
   уравн./коуравн. (`J`=параллельная пара) → pullback/pushout (`J`=коспан/спан).
5. **Сопряжения** левое/правое — через Кан: `G = Ran_F Id`, `F = Lan_G Id` (левый сохраняет
   копределы, правый — пределы).
6. **Монады/комонады** — через Кан/сопряжение: `Codensity f = Ran f f`, `Density f = Lan f f`,
   монада из сопряжения.

Сверху — **сводная таблица** (вручную): `Конструкция | Двойственная | Идея | мини-диаграмма |
Haskell | где в курсе | →подробнее`.

## Соглашение об обозначениях (единое для всех диаграмм/текста)

- **Заглавные латинские** — категории и функторы: `C`, `D`, `F`, `G` (контравариантный — со штрихом, `F′`).
- **Строчные греческие** — естественные преобразования: `η`, `ε`, `α`, `β`, `μ`, `δ`.
- **Строчные латинские** — внутри категории: объекты `a, b, c` и морфизмы `f, g, h`.
- Спец-объекты с устоявшимися именами оставляем как принято: `Ω` (классификатор), терминальный `1`, начальный `0`, `Void`.

## Шаблон раздела пары (скелет — единый для всех)

Каждый раздел самодостаточен и состоит из:
1. **Заголовок** с якорем (для ссылок из таблицы).
2. **Идея/мотивация** — 1–2 предложения, в чём пара и как одно получается из другого
   обращением стрелок (или зеркалом `C^op`).
3. **Мат-определение** обоих членов + **универсальное свойство**.
4. **Парная диаграмма** (SVG, канон Faza 6/Фазы B: светлая палитра, кит стрелок
   mono `↣`/epi `↠`/iso `↔`/nat `⇒`): левый член и правый член бок о бок, видно обращение
   стрелок/конуса. Для лимитной части — форма `J` в центре.
5. **Haskell** обоих членов (исполняемый код, IHaskell).
6. **«В чём двойственность»** — одна строка-резюме.
7. **«Где в курсе»** — ссылки на ноутбуки (pullback↔Toposes, Lan/Ran↔KanExtensions,
   монады↔Monads, алгебры↔AlgebrasCoalgebras и т.д.).

## Реализация: фрагментарность и возобновляемость

- План перечисляет **каждый раздел отдельным независимым шагом** со своим скелетом
  (что наполнять). Разделы не зависят друг от друга по исполнению.
- v1: собрать каркас (титул, мотивация, таблица, setup, NAV) + наполнить образцы по новому
  порядку: моно/эпи (примитив) → Lan/Ran (движок, §2) → одна универсальная конструкция через
  Кан с центральным приёмом (например произв./копроизв. как `J`=дискретная пара). Остальные
  разделы — заглушка-скелет (заголовок + якорь + «TODO по шаблону»), чтобы строка таблицы и
  ссылка уже работали, а наполнение шло потом без потери общей картины.
- Диаграммы рисуются по канону (`DesignShowcase.ipynb` + `normalize_palette.js`), ASCII-only
  в SVG, матсимволы через `&#NNNN;`, кит маркеров стрелок.
